# 因子组合反模式

| 反模式 | 为什么错 | 修复 |
|---|---|---|
| 把所有正分因子直接相加 | 高相关因子会重复押注 | 先算相关矩阵和 top overlap |
| 组合前不统一 horizon / universe | 不同标签混在一起，结果不可解释 | 只组合同口径因子 |
| 用 test 段调权 | 污染最终验证 | train / val 确定权重，test 不可见 |
| 优化器无权重上限 | 单个因子支配组合 | 设置 `abs(w_i) <= 0.35` 等约束 |
| 只和平均单因子比较 | 平均值太弱，容易虚假提升 | 必须和最佳单因子比较 |
| 相关性越低越好 | 低相关但无 alpha 没价值 | 先过质量线，再看低相关 |
| 忽略交易成本 | 组合可能把高换手噪声叠加 | 权重里惩罚 turnover |
| 不做 leave-one-out | 不知道组合是否依赖单个因子 | 逐个去掉因子重评 |
| 负权随意开放 | 可能变成隐含做空或反向押注 | 只有策略允许时才开负权 |
| 组合后不重新评价 | 无法确认复合 Alpha 是否真实提升 | 调 factor-evaluate 完整评价 |

## 危险信号清单

如果出现以下情况，先停下来排查：

- composite score 大幅超过所有单因子，但相关性诊断缺失
- 最大权重大于 50%
- 入选因子之间 max rank corr 大于 0.8
- top-bucket overlap 大于 70%
- 组合表现只在某一年有效
- leave-one-out 去掉一个因子后 score 变负
- turnover 高于所有单因子很多
- 用了 test 段信息决定权重或剔除因子

真正稳健的复合 Alpha 通常是“适度提升 + 更平滑 + 更低回撤”，不是突然出现远超单因子的神奇分数。
